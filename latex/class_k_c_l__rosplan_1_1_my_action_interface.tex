This function works has a callback for the \textquotesingle{}goto\+\_\+waypoint\textquotesingle{} action in PDDL domain file. During the dispatching of the plan, this function get triggers with the occurance of \textquotesingle{}goto\+\_\+waypoint\textquotesingle{} action. It contains the low-\/level controller implementation of \textquotesingle{}goto\+\_\+waypoint\textquotesingle{} action which made the robot in the Gazebo simulation move from one waypoint to another using a action service. Once the robot has reach its desired waypoint location, it calls two services \textquotesingle{}request\+\_\+set\+\_\+orientation\textquotesingle{} and \textquotesingle{}request\+\_\+move\+\_\+arm\textquotesingle{} to collect the hints. \mbox{\Hypertarget{class_k_c_l__rosplan_1_1_my_action_interface_ae90cfd4970c0be72e396181cd1910d2b}\label{class_k_c_l__rosplan_1_1_my_action_interface_ae90cfd4970c0be72e396181cd1910d2b}}

This function update the knowlegde in the ROSPlan KB in two situations. Firstly, when the robot has collected hints but they are inconsistent, incomplete and incorrect. In this case, knowlegde about the visited waypoints will be remove from the KB. This way after replanning robot will visit all the waypoints again and collect new hints. Secondly, when the robot has collect hints that are consistent, complete and correct. In this case all the waypoints will be marked as visited so that the robot directly go to wp0 and print the hint statment.
